# "今天吃什么" 老虎机 H5 应用

## Context
纯前端 H5，老虎机式选饭轮盘 + 一周饮食统计 + AI 点评/建议。LocalStorage 存储，多 AI Provider 可切换（默认国内）。含食物标签分类 + 偏好加权随机。

## 文件结构
```
what-to-eat/
├── index.html            # 单入口 HTML
├── css/
│   └── style.css         # 全部样式 + CSS 动画
└── js/
    ├── app.js            # 入口：Tab 切换、初始化
    ├── slot-machine.js   # 老虎机核心动画
    ├── storage.js        # LocalStorage 封装
    ├── food-manager.js   # 食物列表 CRUD + 标签管理
    ├── tag-service.js    # AI 食物标签服务
    ├── stats.js          # 一周统计可视化
    ├── ai-service.js     # AI API 调用层（多 Provider）
    └── settings.js       # API 设置页 + Prompt 编辑
```

## 实现步骤

### 1. storage.js — 数据层
- Key 设计：
  - `wte_foods`：`[{name, tag}]`
  - `wte_records`：`[{date, food, time}]`
  - `wte_ai_settings`：`{provider, apiKeys, customEndpoint, customModel}`
  - `wte_prompts`：`{achievementPrompt, healthPrompt, tagPrompt, prefixPrompt, pairingPrompt}` ← 5 个可编辑 Prompt
  - `wte_favorites`：`["湖南牛肉粉", "湖南牛肉粉", "螺蛳粉"]`（3 槽位可重复）
  - `wte_prefixes`：`["神仙", "暴走", "灵魂", ...]` ← 趣味前缀列表，可维护
- 首次启动写入默认食物（含预置标签）、默认前缀列表和 5 个默认 Prompt

### 2. index.html + style.css — 页面骨架与样式
- 底部 5 Tab：选饭 / 偏好 / 统计 / 点评 / 设置
- 老虎机视窗：3 项可见、指示线标记中间
- **首页（选饭 Tab）布局从上到下**：
  1. 老虎机 + 选饭按钮
  2. 选中结果展示 + 搭配提示/选择
  3. **最近吃饭成就展示区**：显示最近解锁的 2-3 个成就（成就名 + 稀有度 badge），点击可展开详情
  4. "解锁新成就"按钮（手动触发 AI 生成）
- **偏好 Tab（独立页面）**：
  - 食物按大类分组（面食、粉类、米饭、粥品、小吃、快餐、火锅 …）
  - 每组标题 + 食物方块网格布局
  - 方块可点选高亮，可重复选同一食物（第一次点=第1偏好，第二次=第2偏好，第三次=第3偏好）
  - 最多选 3 个，已选 3 个后其他方块变灰不可选
  - 顶部显示已选偏好："❤️1: 油泼面  ❤️2: 油泼面  ❤️3: 螺蛳粉"
  - 点击已选中的方块取消（释放一个槽位）
  - 显示加权规则说明
- 配色：珊瑚红 + 明黄 + 草绿，活泼卡通风
- 手机适配：max-width 480px、safe-area、44px 最小触控
- 成就存储：`wte_achievements` → `[{name, condition, rarity, date}]`，LocalStorage 持久化

### 3. slot-machine.js — 老虎机核心（偏好加权随机）
- 偏好加权随机只在 spin() 选目标食物时生效，与统计无关
- **趣味前缀系统**：从 storage 读取前缀列表，显示名 = 随机前缀 + 原食物名
- 前缀列表可在设置页手动增删、一键 AI 批量刷新
- **自动填充**：食物不足 15 个时，同食物配不同前缀填充
- **偏好加权随机**：同上算法
- CSS transition + cubic-bezier 自然滚动
- spin() resolve 原始食物名
- rebuild() 供食物/偏好变更后重建

### 3.1 搭配提示功能
- spin() 选中食物后，如果有配 AI → 自动调用 AI 获取搭配建议
- 结果展示在选中食物下方，如：
  "建议搭配：凉皮(+200千卡) · 稀饭(+80千卡)"
  每个搭配可点击选择/取消
- **用户选择搭配后，记录格式变为"肉夹馍+凉皮"**，写入 wte_records 进入统计
- 不选搭配则只记录原始食物名
- 搭配提示使用本地缓存优先，缓存未命中则调 AI
- `wte_pairings`：LocalStorage 缓存 → `{"肉夹馍": [{pairing:"凉皮", reason:"...", calories:200}, ...]}`
- 设置页新增第5个 Prompt：**搭配提示 Prompt（pairingPrompt）**
- 未配 AI 时只显示选中结果，不显示搭配提示

### 4. food-manager.js — 食物管理
- 列表渲染（食物名 + 标签 badge）+ 删除 + 添加
- 至少保留 3 项
- 添加新食物时：自动打标签（有 AI 则调 AI，没有则标"其他"）
- 可手动改标签（下拉选择预置标签列表）
- **"给所有食物打分类标签"按钮**（醒目位置）：
  - 按钮旁提示文字："输入完食物清单后要点一下哦"
  - 一键调用 AI 给所有食物打标签（覆盖旧的）
  - 未配 AI Key 时提示先去设置页配置
  - 打标签中显示 loading 状态
  - 完成后刷新列表展示新标签
- 增删后通知老虎机 rebuild

### 5. tag-service.js — AI 食物标签服务
- 调用 AI API 批量给食物打标签
- 被 food-manager 的"打标签"按钮和设置页的一键按钮共用
- 打标签失败的食物保留原标签不变
- 打标签 Prompt 从 storage 读取（可自定义调试）
- 返回结果解析：AI 返回 JSON 数组 `[{name, tag}]`，匹配回食物列表

### 6. stats.js — 饮食统计
- 纯按实际记录展示，不加权
- 时间范围切换：最近一周 / 最近一个月
- 按食物聚合计数，纯 CSS 横向条形图，按标签大类着色
- 日历概览：每天吃了什么

### 7. ai-service.js — AI 调用层
- Provider：DeepSeek / 通义千问 / Moonshot / OpenAI / Claude / 自定义（国内优先）
- 统一 chat(messages) 接口
- Claude 用 Anthropic 格式，其余 OpenAI 兼容格式

### 8. settings.js — API 设置页 + Prompt 编辑 + 前缀管理
- Provider 选项卡 + Key 输入
- **5 个 Prompt 编辑区**（每个有"恢复默认"按钮）：
  1. **吃饭成就 Prompt（achievementPrompt）**：根据一周饮食数据，参考游戏成就系统（魔兽世界、Dota、三角洲、瓦、燕云十六声等）的成就命名风格，生成食物化成就名。如"面食霸主·碳水战士""连吃三天螺蛳粉·臭味相投""火锅征服者"等
  2. **热量估算+健康建议 Prompt（healthPrompt）**：分两部分 — 先估算一周摄入热量概览，再给出营养均衡的健康饮食建议
  3. **食物打标签 Prompt（tagPrompt）**：给食物列表的每个食物打一个类型标签（如陕西面食、湘味粉面、粤式粥品等），返回 JSON 格式
  4. **趣味前缀生成 Prompt（prefixPrompt）**：参考知名电影、游戏、动漫的梗，生成食物趣味前缀。如"艾泽拉斯""暗黑破坏""赛博朋克""进击的"等
  5. **搭配提示 Prompt（pairingPrompt）**：给选中的食物推荐常见的搭配组合
- **偏好选择**：3 个下拉框选食物（可重复），说明加权规则
- **趣味前缀管理**：
  - 展示当前所有前缀，可手动增删改
  - **一键 AI 批量刷新**：调用 AI 用 prefixPrompt 生成一批新前缀，替换或追加
  - "恢复默认"按钮恢复预置前缀列表
- 保存校验 + Toast

## 偏好加权随机算法（仅在 slot-machine.js 的 spin() 中使用）

### 标签体系设计（三层，从具体到宽泛）
```
第3层-具体标签: 陕西面食、湘味粉面、广西粉面、川味小吃 ...
第2层-大类:    面食、粉类、米饭、粥品、小吃、快餐、火锅 ...
第1层-口味:    辣味、清淡、酸甜、咸香 ...
```
每个食物有一个具体标签，映射到对应大类和口味。映射关系在 storage 中用一个 tagMap 维护。

### 漂移机制
同类判定使用**大类**而非具体标签，这样：
- 偏好兰州牛肉面（具体标签: 西北面食, 大类: 面食）
- "同类"漂移到所有面食大类下的食物：油泼面、肉夹馍、日式拉面 等
- 而不是只漂移到西北面食下的兰州拉面自己

如果某个大类下食物太少（≤2个），则继续向上漂移到**口味维度**：
- 如偏好辣的，就可以漂移到辣味下的其他大类（面食→粉类→小吃），都是辣的

### 算法
```
weightedRandom(foods, favorites):
  无偏好 → 均匀随机

  统计偏好食物频次:
    例1: [兰州牛肉面×3] → 唯一1个
    例2: [油泼面×2, 螺蛳粉×1] → 唯一2个

  === 场景A: 3个都选同一食物 ===
    该食物自身: 60%
    同大类不同食物(漂移): 20%
      - 如偏好兰州牛肉面(面食类)→ 同属面食的油泼面、肉夹馍等
      - 如果面食类下只有兰州牛肉面自己 → 向上漂移到同口味(辣味)的其他大类
    其余: 20%

  === 场景B: 2同一异 ===
    重复食物: 40%
    另一个偏好: 20%
    同大类漂移(两个偏好的大类都算): 20%
    其余: 20%

  === 场景C: 3个不同 ===
    3个偏好各: 20%×3 = 60%
    同大类漂移: 20%
    其余: 20%

  漂移查找顺序:
    1. 同具体标签 + 不同食物 → 2. 同大类 + 不同食物 → 3. 同口味 + 不同食物
    优先在第1层找，找不到足够候选则向上一层扩展
```

## 验证方式
- 老虎机滚动 + 前缀趣味展示
- 设 3 偏好为同一食物，多次 spin 确认加权效果
- 手动改标签、一键 AI 打标签
- 手动增删前缀、一键 AI 批量刷新前缀
- Prompt 编辑后点评/建议/打标签/生成前缀都使用新 Prompt
- 统计页实际记录展示正确（一周/一个月切换）
